# ADR (Architecture Decision Records)

Назначение: хранить краткие записи архитектурных решений, чтобы:
- было понятно, **почему** выбран конкретный вариант,
- можно было ревьюить и пересматривать решения при изменении предпосылок,
- было к чему привязывать требования, тесты и evidence.

Шаблон: `docs/decisions/ADR-TEMPLATE.md`.

Формат имени файла: `ADR-XXX_<short_name>.md` (как у `docs/design-notes/DN-XXX_<short_name>.md`).

## /obsolete
Папка `docs/decisions/obsolete/` — для устаревших ADR: решения, которые больше неактуальны, заменены новыми ADR/Design Notes или признаны незначимыми для текущей системы.

Правило: при переносе документа в `obsolete/` добавлять в начале файла пометку о причине и ссылку на актуальный документ (если есть).
