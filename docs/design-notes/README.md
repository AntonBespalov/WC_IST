# Design Notes (DN)

Назначение: хранить развернутые проектные записки (Design Notes), которые описывают:
- контекст и мотивацию,
- варианты и компромиссы,
- интерфейсы и ограничения,
- риски и план доказательств (tests/proof).

Шаблон: `docs/design-notes/DN-TEMPLATE.md`.

## /obsolete
Папка `docs/design-notes/obsolete/` — для устаревших DN: документы, которые больше неактуальны, заменены новыми DN/ADR или признаны незначимыми для текущей системы.

Правило: при переносе документа в `obsolete/` добавлять в начале файла пометку о причине и ссылку на актуальный документ (если есть).
